

## Add Currency to Risk Products Payment Amount

Currently, in `src/data/regional360ViewData.ts` line 435, the payment amount is generated by calling `formatCurrency(premium, config)` and then stripping the currency symbol with `.replace(config.currencySymbol + " ", "")`. This means the displayed value has no currency prefix.

### Fix

**File:** `src/data/regional360ViewData.ts` (line 435)

Remove the `.replace(...)` call so the full formatted currency string (e.g., "R 2,500.00", "A$ 3,200.00", "$4,100.00") is kept intact for all jurisdictions.

**Before:**
```ts
paymentAmount: formatCurrency(premium, config).replace(config.currencySymbol + " ", ""),
```

**After:**
```ts
paymentAmount: formatCurrency(premium, config),
```

This single-line change ensures the currency symbol appears for ZA, AU, CA, GB, and US jurisdictions automatically, since `formatCurrency` already handles jurisdiction-specific symbols.
